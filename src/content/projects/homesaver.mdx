<h3>Overview</h3>

I built this Chrome extension to solve a problem on the YouTube homepage, a misclick or refresh can instantly change recommendations and make it easy to lose a video you intended to watch. This extension automatically saves video links when ever the user visits the homepage.

<h3>Notable Tech Used</h3>

<h4>Svelte</h4>

- Svelte with Vite was used for the popup/main window
- Selected Svelte over React for its compile time architecture, producing smaller bundles and faster startup for a lightweight Chrome extension popup.
- used Svelte's component model and built-in reactivity to simplify UI updates
- Utilized Svelte’s built-in fade transition for lightweight UI animations without additional libraries.
- Chose Svelte to learn a a new Framework other then React

<h4>Content Scripts</h4>

- Content scripts to obtain data from internal data layer (ytInitialData), instead of relying on fragile DOM scraping
- Handled YouTube's single-page navigation by listening for navigation events with MutationObserver fallback, ensuring data capture across route changes
- Used the chrome.storage.local API, storing up to 4 sets of 6 videos with automatic cleanup when limits exceeded
- Prevent storing redundant data when users refresh the same homepage
